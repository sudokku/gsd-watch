# Quick Task 260325-398: Phase Title Stays White+Bold When Navigating Summary

**One-liner:** Phase tree active-state highlighting extended to child plan rows via `IsPhaseActive` helper, with badge lines inheriting the same active/dimmed/default styling.

## What Was Done

Fixed two TUI rendering bugs in the phase tree (`internal/tui/tree/view.go`):

1. **Phase title highlighting bug**: Previously, the phase title only highlighted when `i == t.cursor`. If the cursor was on a child plan row, the phase title reverted to its default/dimmed style. Now the `IsPhaseActive` helper checks whether the cursor is on the phase row OR any consecutive `RowPlan` child rows beneath it.

2. **Badge styling bug**: Badge tags (`[disc]`, `[rsrch]`, etc.) always rendered as plain unstyled text regardless of phase state. Now badge lines use the same three-way switch: `highlightStyle` when active, `PendingStyle` when completed+inactive, plain text otherwise.

## Implementation Details

**`IsPhaseActive(rows []Row, phaseRowIdx int) bool`** — exported method on `TreeModel`. Walks forward from `phaseRowIdx + 1`, treating consecutive `RowPlan` rows as children until a non-RowPlan row or end of slice is reached. Returns `true` if `t.cursor` falls in that range or equals `phaseRowIdx`.

**RowPhase rendering changes:**
- Compute `phaseActive := t.IsPhaseActive(rows, i)` once before the name-parts loop
- Name text: `case phaseActive` replaces `case i == t.cursor`
- Prefix string (expand indicator + icon): uses `highlightStyle` when active, `PendingStyle` when dimmed, plain otherwise
- Badge line: new `switch` block applies `highlightStyle`/`PendingStyle`/plain instead of always plain
