**Test setup pattern:** Each test sets `DebugOut = &buf` (bytes.Buffer) at start and defers `DebugOut = nil` to restore package state. Tests use `strings.Contains` on `buf.String()` for assertions — NEVER match exact timestamps (they change every second, see RESEARCH.md Pitfall 5).

**Test implementations:**

1. **TestDebugSilentByDefault** — Verify `DebugOut` is nil. Call `debugf("test", "hello")`. No panic. No output anywhere.

2. **TestDebugPhaseDir** — Create a temp dir with a phase dir `01-test-phase/` containing a minimal `01-01-PLAN.md` with valid YAML frontmatter (use `"` + three dashes + `\nstatus: pending\n` + three dashes + `\n"` as the file content string in Go). Call `parsePhases(phasesDir, map[int]string{1: "test phase"}, 0, 0)` with `DebugOut = &buf`. Assert `buf.String()` contains `phase_dir:`, `num=1`, and `name="Phase 1: test phase"`.

3. **TestDebugPlan** — Same fixture as above. Assert output contains `plan:`, `status=`, `title=`, and `wave=`.

4. **TestDebugPlanError** — Create a phase dir with a PLAN.md containing invalid YAML frontmatter. Use a Go string like: `"` + three dashes + `\nstatus: [invalid yaml\n` + three dashes + `\n"`. This causes yaml.Unmarshal to fail. Call `parsePhases`. Assert output contains `plan_error:` and `err=`.

5. **TestDebugBadge** — Create a phase dir `01-test/` with an empty `01-CONTEXT.md` file (badge file). Call `parsePhases`. Assert output contains `badge:` and `discussed` (the badge constant for CONTEXT.md).

6. **TestDebugCacheHIT** — Create a planning root with setupTestPlanning helper (or manual fixture). Create a `NewCache(root)`, call `ParseFull()`, then call `Update(statePath)` without changing the file. Assert output contains `cache:` and `HIT`.

7. **TestDebugCacheMISS** — Same setup, but modify the target file's content (write new data with os.WriteFile) before calling `Update()`. Assert output contains `cache:` and `MISS`. Note: the file mtime must actually change (different from ParseFull time). Writing new content is sufficient since os.WriteFile updates mtime.

8. **TestDebugFormat** — Set `DebugOut = &buf`, call `debugf("test_event", "key=%d", 42)`. Assert output matches regex `\[debug \d{2}:\d{2}:\d{2}\] test_event: key=42\n`.

**Important:** Each test must reset `DebugOut = nil` in a defer to avoid polluting other tests. Use `t.Helper()` for any shared fixture setup functions.
