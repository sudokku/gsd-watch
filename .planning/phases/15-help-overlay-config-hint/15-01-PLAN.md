# Plan 15-01: Add config path and theme name to help overlay

## Goal

Extend `helpView` in `internal/tui/app/model.go` to render a "Config" section showing the tilde-abbreviated config file path and the active theme name.

## Context

Phase 13 established `config.ConfigPath` (the XDG-relative path constant) and the sentinel `cfg.Theme = ""` meaning "default". Phase 14 confirmed `ResolveTheme("")` returns the "default" canonical name. This phase surfaces both values in the existing `?` overlay so users can discover and edit their settings without consulting docs.

All changes are additive to two existing files: `model.go` (implementation) and `model_test.go` (tests). No new packages or files are needed.

## Tasks

### Task 1: Extend `helpView` signature and render Config section

**Files:**
- `internal/tui/app/model.go`

**Action:**

1. Change the `helpView` signature from:
   ```go
   func helpView(width int, noEmoji bool) string
   ```
   to:
   ```go
   func helpView(width int, noEmoji bool, configPath, themeName string) string
   ```
   This follows the same extension pattern used in Phase 10 when `noEmoji bool` was added.

2. Append a `Config` section to `helpText` inside `helpView`, after the `phaseStages` block and before `press q or esc to close`:
   ```
   Config
   Config:  <configPath>
   Theme:   <themeName>
   ```
   Use `fmt.Sprintf` (or string concatenation) to interpolate `configPath` and `themeName`. The label/value pairs should be flush-left ‚Äî no special alignment needed.

3. Update the `View()` call site (currently line 339) to compute and pass both values inline before calling `helpView`:
   ```go
   if m.helpVisible {
       home, _ := os.UserHomeDir()
       cfgPath := strings.Replace(config.ConfigPath, home, "~", 1)
       // Wait ‚Äî config.ConfigPath is a relative constant, not an absolute path.
       // Construct the absolute path first: filepath.Join(home, config.ConfigPath)
       // then tilde-abbreviate it.
       absPath := filepath.Join(home, config.ConfigPath)
       cfgPath := strings.Replace(absPath, home, "~", 1)
       themeName := m.cfg.Theme
       if themeName == "" {
           themeName = "default"
       }
       return helpView(m.width, !m.cfg.Emoji, cfgPath, themeName)
   }
   ```
   Add `"os"`, `"path/filepath"`, and `"strings"` to the import block if not already present. The `config` package is already imported.

**Note:** `config.ConfigPath` is the constant `".config/gsd-watch/config.toml"` (relative). The absolute path is `filepath.Join(home, config.ConfigPath)`. Tilde-abbreviation replaces `home` with `~` in the absolute path, yielding `~/.config/gsd-watch/config.toml`.

### Task 2: Update tests for the new `helpView` signature and assert Config section content

**Files:**
- `internal/tui/model_test.go`

**Action:**

`helpView` is a package-level function tested indirectly via `m.View()` after pressing `?`. No direct `helpView(...)` call sites exist in the test file ‚Äî the signature change has no mechanical breakage there. However, two additions are required:

1. **Add a test for DISC-01** ‚Äî config path appears in the overlay:
   ```go
   // TestHelpOverlay_ContainsConfigPath: '?' overlay shows the config file path (DISC-01).
   func TestHelpOverlay_ContainsConfigPath(t *testing.T) {
       m := newTestModel()
       m, _ = updateModel(m, tea.WindowSizeMsg{Width: 80, Height: 24})
       m, _ = updateModel(m, tea.KeyMsg{Type: tea.KeyRunes, Runes: []rune{'?'}})
       view := m.View()
       if !strings.Contains(view, "Config:") {
           t.Error("expected help overlay to contain 'Config:' label")
       }
       if !strings.Contains(view, "config.toml") {
           t.Error("expected help overlay to contain 'config.toml' in config path")
       }
   }
   ```

2. **Add a test for DISC-02** ‚Äî theme name appears in the overlay:
   ```go
   // TestHelpOverlay_ContainsThemeName: '?' overlay shows the active theme name (DISC-02).
   func TestHelpOverlay_ContainsThemeName(t *testing.T) {
       m := newTestModel()
       m, _ = updateModel(m, tea.WindowSizeMsg{Width: 80, Height: 24})
       m, _ = updateModel(m, tea.KeyMsg{Type: tea.KeyRunes, Runes: []rune{'?'}})
       view := m.View()
       if !strings.Contains(view, "Theme:") {
           t.Error("expected help overlay to contain 'Theme:' label")
       }
       if !strings.Contains(view, "default") {
           t.Error("expected help overlay to contain 'default' theme name when cfg.Theme is empty")
       }
   }
   ```

   `newTestModel()` initialises with `config.Defaults()` which sets `Theme: ""`, so `themeName` will be `"default"` ‚Äî the assertion is correct.

3. Verify that existing tests (`TestHelpOverlay_ContainsPhaseStages`, `TestNoEmoji_HelpOverlay_ASCIIBadges`, and `TestHelpOverlay_OpenClose`) still compile and pass ‚Äî no direct `helpView` call sites in the test file means no mechanical changes needed for them.
